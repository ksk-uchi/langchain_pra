## 役割と原則

あなたは**サブエージェント（コーダー）**です。以下の手順を**そのまま実行**してください。設計判断・リファクタリング・スコープ外の改善は一切行わないでください。既存の `commands/registry/` には一切手を加えないこと。

---

## 作業概要

`python commands/query/main.py --collection=corpus --n=5 --query=hogehoge` で動作するクエリコマンドを実装する。

- `--collection` 省略時: `--query` をそのまま LLM に問い合わせる（RAG なし）
- `--collection` 指定時: `--query` をベクトル化して ChromaDB から上位 `--n` 件を取得し、その内容をコンテキストとして LLM に問い合わせる（RAG あり）

## 実装手順

### Step 1: ファイルの用意

`commands/query/__init__.py` を空ファイルとして作成する。

### Step 2: `commands/query/main.py` の実装

#### 2-1. インポートと定数定義

`registry/main.py` と同じパターンで `.env` をロードする。

```python
from pathlib import Path

_REPO_ROOT = Path(__file__).resolve().parent.parent.parent
load_dotenv(_REPO_ROOT / "envs" / "local.env")
```

定数として以下を定義する（`registry/main.py` の `CHROMA_PERSIST_DIR` と同じ値にすること）:

- `EMBEDDING_MODEL_ID = os.getenv("EMBEDDING_MODEL_ID", default="gemini-embedding-001")`
- `QUERY_MODEL_ID = os.getenv("QUERY_MODEL_ID", default="gemini-2.5-flash")`
- `CHROMA_PERSIST_DIR = "chroma_db"`

#### 2-2. click コマンド定義

以下の 3 オプションを持つ `query` コマンドを定義する:

| オプション | 型 | 必須/省略可 | デフォルト | 説明 |
|---|---|---|---|---|
| `--query` | `str` | 必須 | — | LLM へ問い合わせるテキスト |
| `--collection` | `str` | 省略可 | `None` | ChromaDB のコレクション名 |
| `--n` | `int` | 省略可 | `5` | RAG で取得するドキュメント数 |

#### 2-3. LLM の初期化

`langchain_google_genai` の `ChatGoogleGenerativeAI` を使い、`model=QUERY_MODEL_ID` で初期化する。

#### 2-4. `--collection` 省略時（RAG なし）の処理

`ChatPromptTemplate` で「ユーザーの質問をそのまま回答する」シンプルなプロンプトを作り、`StrOutputParser` を組み合わせた LCEL チェーンで `query` を渡して結果を `click.echo` する。

#### 2-5. `--collection` 指定時（RAG あり）の処理

以下の流れで実装する:

1. `GoogleGenerativeAIEmbeddings(model=f"models/{EMBEDDING_MODEL_ID}")` を初期化
2. `chromadb.PersistentClient(path=CHROMA_PERSIST_DIR)` で ChromaDB に接続
3. `langchain_chroma.Chroma` を `client=` と `collection_name=` と `embedding_function=` を渡して初期化し、既存コレクションに接続する
   - `Chroma(client=client, collection_name=collection, embedding_function=embeddings)`
4. `vectorstore.as_retriever(search_kwargs={"k": n})` でリトリーバーを作成
5. 以下の LCEL チェーンを組む:
   - `context` キー: リトリーバーの結果を結合するヘルパー関数（`"\n\n".join(doc.page_content for doc in docs)`）
   - `question` キー: `RunnablePassthrough()` でそのまま通す
   - プロンプトテンプレート（下記参照）→ LLM → `StrOutputParser`
6. `chain.invoke(query)` の結果を `click.echo` する

**RAG 用プロンプトテンプレート（この内容を使うこと）:**

```
以下のコンテキストを参考に、質問に答えてください。

コンテキスト:
{context}

質問: {question}
```

#### 2-6. `if __name__ == "__main__":` エントリポイント

`registry/main.py` と同じパターンで追加する。
