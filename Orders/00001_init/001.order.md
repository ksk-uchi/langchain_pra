## 背景

- LangChain を使ってどんなことが出来るか、また使う際どんな実装にするとよいかが理解できるように実験したい
- LangChain で RAG を試すが構成は LangChain + ChromaDB で embedding は Google の `gemini-embedding-001` を用いる
  - ChromaDB から取得した top N 件のコンテキストとクエリを問い合わせる先は `gemini-2.5-flash` とする
- 今回の作業ではコーパスを ChromaDB に登録できるコマンドの作成までを行う

## 要件

- `commands/` ディレクトリを作り、その中に `registry/` のようなコマンドディレクトリを作る
  - `python commands/registry/main.py --path=corpus/hogehoge.txt` のようにコマンドを実行する
  - コマンドは Python で実装する。ライブラリは click を使用する
  - `Orders/00001_init/001.order.md` でつくる生成物は `registry` コマンドを作り `ChromaDB` にコーパスを保存することが出来るようにする
    - `python commands/registry/main.py --path=corpus/hogehoge.txt` のように実行すると `hogehoge.txt` がチャンク化されてそれぞれ ChromaDB に格納される
    - `--path` はディレクトリも指定できる。その場合はディレクトリに含まれる全ファイルを取り込む
    - ChromaDB にはベクトルの他、実際のテキストや metadata としてファイル名、embedding に使った LLM の ID が入る
    - ChromaDB に格納する際のデータの ID は `(file path|hash)--(chunk size)--(chunk id)--(llm id)` という ID を採用する
- リポジトリのトップに `envs/` を作り、その中に `local.env.sample` というファイルを作る
  - Google API Key は環境変数 `GOOGLE_API_KEY` を参照するように実装する (`local.env.sample` には `GOOGLE_API_KEY=(your google api key)` としておく)
  - `local.env` というファイルも作成するがこちらは `.gitignore` に記載し、バージョン管理しないようにする
- コードのリンター兼フォーマッターは `ruff` を採用する。
- 型チェックは `mypy` を採用する
- `pre-commit` をインストールし、 `git commit` する際に `ruff` と `mypy` のチェックが走るようにする

## AI からの質問と、私からの回答

```
  1. チャンクサイズ
  チャンクのサイズ（文字数）はどのくらいをデフォルトにしますか？また --chunk-size のようなオプションで実行時に変更できるようにしますか？

  2. チャンクオーバーラップ
  チャンク間のオーバーラップ（overlap）は設けますか？（RAGの品質向上のため一般的には設けますが、要件に記載がなかったため確認です）

  3. ディレクトリ指定時の対象ファイル形式
  --path にディレクトリを指定した場合、どの拡張子のファイルを対象にしますか？（例: .txt のみ、.txt と .md、全ファイルなど）

  4. ChromaDB の保存先
  ChromaDB のデータはリポジトリ内のどこに保存しますか？（例: ./chroma_db/）また --db-path のようなオプションで変更可能にしますか？

  5. ID フォーマットの詳細
  (file path)-(chunk size)-(chunk id)-(llm id) の file path 部分は、どの形式を想定していますか？
  - フルパス or 相対パス？
  - パス区切り文字（/）はそのまま含める？それとも別の文字（_ など）に置換？

  6. 既存データの上書き
  同じ ID のデータが ChromaDB にすでに存在する場合、どう扱いますか？（上書き / スキップ / エラー終了）

  7. local.env の自動読み込み
  python-dotenv などを使って envs/local.env を自動で読み込む実装にしますか？それともユーザーが事前に export GOOGLE_API_KEY=... する前提ですか？

  8. ChromaDB のコレクション名
  ChromaDB に格納するコレクション名は固定（例: corpus）ですか？それとも --collection オプションで指定可能にしますか？
```

1. チャンクサイズ
   - 500 トークンとしてください
2. チャンクオーバーラップ
   - オーバーラップは 10% にあたる 50 トークンとしてください
3. ディレクトリ指定時の対象ファイル形式
   - 対象ファイルは `.txt` と `.md` です
4. ChromaDB の保存先
   - `chroma_db/` に保存してください
   - バージョン管理したくないので `.gitignore` に追記してください
5. ID フォーマットの詳細
   - `[file path hash]--500--00001--gemini-embed-001` のようにしてください
   - ファイルパスは相対パスです
   - ファイルパスを直接使うのではなくハッシュ化するようにしてください
   - セパレータは `-` ではなく `--` とするようにしてください
   - チャンク ID は 5 桁で左ゼロパディングしてください
6. 既存データの上書き
   - 同じ ID のデータがある場合は上書きでお願いします
7. local.env の自動読み込み
   - `python-dotenv` を使ってください
8. ChromaDB のコレクション名
   - `--collection` で指定可能としてください。デフォルトは `corpus` でお願いします
