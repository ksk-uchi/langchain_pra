# サブエージェント向け作業手順書: registry コマンドの実装

## 概要

LangChain + ChromaDB を使ったコーパス登録コマンド `commands/registry/main.py` を実装する。
テキストファイルをチャンク化して Google Gemini embedding でベクトル化し、ChromaDB に upsert する CLI ツール。

**作業後にコミットを1つ作成すること。UIや動作確認（実際にコマンドを実行してAPIを叩く確認）はユーザーが行うため、サブエージェントは静的チェック（ruff, mypy）のみ実行すること。**

### ステップ 3: `.pre-commit-config.yaml` を作成

リポジトリルートに `.pre-commit-config.yaml` を作成すること。
ruff のリビジョンは `uv run ruff --version` で確認したバージョン番号を `v{バージョン}` 形式で設定すること（例: ruff 0.9.6 なら `rev: v0.9.6`）。

```yaml
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v{ruff のバージョン番号}  # uv run ruff --version で確認して置き換えること
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format
  - repo: local
    hooks:
      - id: mypy
        name: mypy
        entry: uv run mypy
        language: system
        types: [python]
        pass_filenames: true
```

### ステップ 6: `commands/registry/main.py` を実装

以下のファイル・ディレクトリを作成すること。

**`commands/__init__.py`** — 空ファイル

**`commands/registry/__init__.py`** — 空ファイル

**`commands/registry/main.py`** — 以下の通りに実装すること。

```python
"""Registry command: ingest corpus files into ChromaDB."""
from __future__ import annotations

import hashlib
from pathlib import Path

import chromadb
import click
from dotenv import load_dotenv
from langchain_google_genai import GoogleGenerativeAIEmbeddings
from langchain_text_splitters import RecursiveCharacterTextSplitter

_REPO_ROOT = Path(__file__).resolve().parent.parent.parent
load_dotenv(_REPO_ROOT / "envs" / "local.env")

EMBEDDING_MODEL_ID = "gemini-embedding-001"
CHUNK_SIZE = 500
CHUNK_OVERLAP = 50
SUPPORTED_EXTENSIONS: frozenset[str] = frozenset({".txt", ".md"})
CHROMA_PERSIST_DIR = "chroma_db"


def _collect_files(path: Path) -> list[Path]:
    """ファイルならそのまま、ディレクトリなら対象拡張子のファイル一覧を返す。"""
    if path.is_file():
        if path.suffix in SUPPORTED_EXTENSIONS:
            return [path]
        return []
    return sorted(
        p for p in path.rglob("*") if p.is_file() and p.suffix in SUPPORTED_EXTENSIONS
    )


def _file_path_hash(relative_path: str) -> str:
    """相対パスの SHA256 ハッシュ（先頭 16 文字）を返す。"""
    return hashlib.sha256(relative_path.encode()).hexdigest()[:16]


def _build_doc_id(file_hash: str, chunk_idx: int) -> str:
    """ChromaDB 格納用 ID を構築する。

    形式: {file_path_hash}--{chunk_size}--{chunk_id:05d}--{embedding_model_id}
    例:   a1b2c3d4e5f60718--500--00001--gemini-embedding-001
    """
    return f"{file_hash}--{CHUNK_SIZE}--{chunk_idx:05d}--{EMBEDDING_MODEL_ID}"


def _ingest_file(
    file_path: Path,
    splitter: RecursiveCharacterTextSplitter,
    embeddings: GoogleGenerativeAIEmbeddings,
    collection: chromadb.Collection,  # type: ignore[type-arg]
) -> int:
    """1 ファイルを読み込み、チャンキングして ChromaDB に upsert する。

    Returns:
        登録したチャンク数。
    """
    text = file_path.read_text(encoding="utf-8")
    chunks = splitter.split_text(text)
    if not chunks:
        return 0

    cwd = Path.cwd().resolve()
    try:
        relative_path = str(file_path.resolve().relative_to(cwd))
    except ValueError:
        relative_path = str(file_path.resolve())

    file_hash = _file_path_hash(relative_path)

    ids = [_build_doc_id(file_hash, i) for i in range(len(chunks))]
    metadatas = [
        {"file_name": file_path.name, "embedding_model": EMBEDDING_MODEL_ID}
        for _ in chunks
    ]
    vectors = embeddings.embed_documents(chunks)

    collection.upsert(
        ids=ids,
        embeddings=vectors,  # type: ignore[arg-type]
        documents=chunks,
        metadatas=metadatas,  # type: ignore[arg-type]
    )
    return len(chunks)


@click.command()
@click.option(
    "--path",
    "path_str",
    required=True,
    help="インデックス対象のファイルまたはディレクトリのパス。",
)
@click.option(
    "--collection",
    default="corpus",
    show_default=True,
    help="ChromaDB コレクション名。",
)
def registry(path_str: str, collection: str) -> None:
    """コーパスをチャンク化して ChromaDB に登録する。"""
    target = Path(path_str)
    if not target.exists():
        raise click.BadParameter(
            f"パスが存在しません: {target}", param_hint="'--path'"
        )

    files = _collect_files(target)
    if not files:
        click.echo("対象ファイルが見つかりませんでした。(.txt / .md のみ対象)")
        return

    embeddings = GoogleGenerativeAIEmbeddings(model=f"models/{EMBEDDING_MODEL_ID}")
    splitter = RecursiveCharacterTextSplitter.from_tiktoken_encoder(
        chunk_size=CHUNK_SIZE,
        chunk_overlap=CHUNK_OVERLAP,
    )

    client = chromadb.PersistentClient(path=CHROMA_PERSIST_DIR)
    col = client.get_or_create_collection(name=collection)

    total_chunks = 0
    for f in files:
        count = _ingest_file(f, splitter, embeddings, col)
        click.echo(f"  {f}: {count} チャンク登録")
        total_chunks += count

    click.echo(
        f"\n完了: {len(files)} ファイル, {total_chunks} チャンク を "
        f"'{collection}' コレクションに登録しました。"
    )


if __name__ == "__main__":
    registry()
```

#### `# type: ignore` コメントの説明

- `chromadb.Collection[type-arg]` — chromadb の `Collection` はジェネリック型だが型引数の指定が不要なため
- `embeddings=vectors` と `metadatas=metadatas` — chromadb の型スタブと LangChain が返す型の微妙な不一致を抑制するため

mypy エラーが上記以外に出た場合は、エラーの内容を確認し、外部ライブラリ起因であれば `# type: ignore[xxx]` を追加して解消すること。

```bash

```

### ステップ 9: コミット

```bash
git add commands/ envs/ .pre-commit-config.yaml .gitignore pyproject.toml
git commit -m "feat: add registry command to ingest corpus into ChromaDB"
```

コミット時に pre-commit フックが自動実行される。フックがエラーになった場合はその内容を修正し、再コミットすること（`--no-verify` は使用しないこと）。
