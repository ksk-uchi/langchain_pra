# サブエージェント向け作業手順書: deptry 設定修正と依存関係の再追加

## 概要

deptry の DEP002（宣言済みだが直接 import していない依存の警告）を無効化し、
前回誤って削除された `langchain`、`langchain-chroma`、`tiktoken` を依存関係に再追加する。

**静的チェック（ruff, mypy, deptry）を実行してエラーがないことを確認したうえで、1コミット作成すること。**

---

## 完成物チェックリスト

- [ ] `pyproject.toml` の `dependencies` に `langchain`、`langchain-chroma`、`tiktoken` が追加されている
- [ ] `pyproject.toml` に `[tool.deptry]` セクションが追加され `ignore = ["DEP002"]` が設定されている
- [ ] `uv run deptry .` がエラーなしで通る
- [ ] `uv run ruff check commands/` がエラーなしで通る
- [ ] `uv run mypy commands/` がエラーなしで通る
- [ ] コミットが1つ作成されている

---

## 作業手順

### ステップ 1: 依存関係の再追加

```bash
uv add langchain langchain-chroma tiktoken
```

### ステップ 2: `pyproject.toml` に deptry 設定を追加

`pyproject.toml` の末尾に以下を追記すること。

```toml
[tool.deptry]
ignore = ["DEP002"]
```

### ステップ 3: 静的チェックを実行

```bash
uv run deptry .
uv run ruff check commands/
uv run mypy commands/
```

エラーがあれば修正すること。

### ステップ 4: コミット

```bash
git add pyproject.toml uv.lock
git commit -m "fix: re-add removed dependencies and disable deptry DEP002"
```

コミット時に pre-commit フックが自動実行される。フックがエラーになった場合はその内容を修正して再コミットすること（`--no-verify` は使用しないこと）。
