# サブエージェント向け作業手順書: langchain-text-splitters 追加 & deptry 導入

## 概要

`commands/registry/main.py` で使用している `langchain_text_splitters` が `pyproject.toml` に未宣言のため追加する。
あわせて未宣言依存を検出する `deptry` を dev 依存に追加し、pre-commit フックに組み込む。

**静的チェック（ruff, mypy, deptry）を実行してエラーがないことを確認したうえで、1コミット作成すること。**

---

## 完成物チェックリスト

- [ ] `pyproject.toml` の `dependencies` に `langchain-text-splitters` が追加されている
- [ ] `pyproject.toml` の `[dependency-groups] dev` に `deptry` が追加されている
- [ ] `.pre-commit-config.yaml` に `deptry` のローカルフックが追加されている
- [ ] `uv run deptry .` がエラーなしで通る
- [ ] `uv run ruff check commands/` がエラーなしで通る
- [ ] `uv run mypy commands/` がエラーなしで通る
- [ ] コミットが1つ作成されている

---

## 作業手順

### ステップ 1: 依存関係の追加

```bash
uv add langchain-text-splitters
uv add --dev deptry
```

### ステップ 2: `.pre-commit-config.yaml` に deptry フックを追加

既存の `.pre-commit-config.yaml` の `repos:` リストの末尾に以下を追記すること。

```yaml
  - repo: local
    hooks:
      - id: deptry
        name: deptry
        entry: uv run deptry
        language: system
        pass_filenames: false
        args: ["."]
```

### ステップ 3: 静的チェックを実行

```bash
uv run deptry .
uv run ruff check commands/
uv run mypy commands/
```

エラーがあれば修正すること。

### ステップ 4: コミット

```bash
git add pyproject.toml uv.lock .pre-commit-config.yaml
git commit -m "chore: add langchain-text-splitters and deptry for dependency checking"
```

コミット時に pre-commit フックが自動実行される。フックがエラーになった場合はその内容を修正して再コミットすること（`--no-verify` は使用しないこと）。
